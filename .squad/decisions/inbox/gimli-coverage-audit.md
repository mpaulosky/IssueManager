### Why These Priorities?

#### 🔴 Phase 1 (Handlers) — Highest Impact
- **Handlers contain business logic** (validation, repository calls, error handling)
- **Most lines of code** in the codebase
- **Highest risk** — bugs here affect end users directly
- **Already have 3 handler tests** — patterns are established, easy to replicate

#### 🔴 Phase 2 (Mappers) — High Value
- **Data transformation bugs are silent** — they don't throw errors, just corrupt data
- **Mappers are pure functions** — easy to test (no mocking needed)
- **Small test files** — 10-20 test cases each

#### 🟡 Phase 3 (Repos) — Medium Value
- **Already covered by integration tests** (MongoDB + TestContainers)
- Unit tests only add value if repos contain business logic (query construction, filtering, etc.)
- Low lines-of-code (most logic is MongoDB driver calls)

#### 🟢 Phase 4 (Abstractions) — Low Value
- **`Result` and `Result<T>` are simple wrappers** — low bug risk
- **Utilities are typically static helpers** — covered incidentally
- Only needed if aiming for 95%+ coverage
